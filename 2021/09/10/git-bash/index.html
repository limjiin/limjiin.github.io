Shell Command

  • cd : 상하위 폴더로 이동
  • mkdir : 폴더 생성
  • pwd
  • touch : 파일 생성
  • mv 폴더명: 폴더로 이동
  • mv 파일명 파일명 : 파일명 변경
  • cp : 복사
  • rm : 삭제
  • rm -rf : 완전 삭제
  • chmod
  • cat : 파일 내용 확인
  • vi : 파일 내용 수정

Recap - Vim command

  • h j k l - left, down, up, right
  • i : insert
  • v : visual
  • ESC : back to normal mode
  • d : delete
  • dd : delete a line
  • y : yank
  • yy : yank a line
  • p : paste
  • u : undo
  • a : append
  • A : append from end of line
  • o : open line(under)
  • O : open line(upper)
  • H : move to the top of the screen
  • L : move to the bottom of the screen

Command mode

  • :q : quit
  • :q! : quit discarding all changes
  • :w : write
  • :wq : write and quit
  • {number} : jump to {number}th line

Repo

  • mkdir first-repo && cd first repo

  • git init

  • git remote add origin https://github.com/{username}/{reponame}.git

  • touch README.md

  • git add README.md

  • git commit -m “docs: Create README.md”

  • git push -u origin master/main

  • make some change on README.md

  • git add

  • git commit

  • git push origin master/main

Commit Convention

  • 제목은 50자 이내

  • 제목과 내용 사이 한 칸

  • prefix를 사용하여 한 눈에 커밋의 용도를 알기 쉽게 한다.

  • feat : features : 기능 개발 관련

  • docs : documentations

  • conf : configurations : 환경 설정 관련

  • test : test

  • fix : bug-fix : 오류 개선 혹으 ㄴ버그 패치

  • refactor : refactoring

  • ci : Continuous Integration

  • build : Build

  • perf : Performance

커밋메시지참고

git clone

  • git clone {repo address}
  • git add
  • git commit
  • git push

Branch

  • 분기점을 생성하여 독립적으로 코드를 변경할 수 있도록 도와주는 모델

  • master, develop

  • git branch : available local branch

  • git branch -r : available remote branch

  • git branch -a : available all branch

  • git branch stem : create

  • git checkout stem : checkout

  • git checkout -b new-stem : create & checkout

  • git commit -a -m ‘edit readme.md’ : make changes inside

  • git checkout master : make changes inside

  • git merge stem : merge

  • git branch -D stem : delete

  • git push origin stem : push with specified remote branch

  • git diff master/main stem : see the difference between two branches

Undoing

  • git checkout – . or git checkout – {filename}

Unstaging

  • git reset HEAD {filename}

Unstaging and Remove

  • git rm -f {filename}

Edit latest commit

  • git commit –amend

Edit prior commit

  • git rebase -i

abort rebase

  • git rebase –abort

Complete rebase

  • git rebase –continue

Reset commit : Revert

  • git revert –no-commit HEAD~3
  • git commit
  • git push origin
  • 잘못하기 전 과거로 돌아가 최신을 유지하면서 되돌렸다는 이력을 commit으로 남겨 모든 팀원이 이 사항을 공유하고 주지시킬 수 있음

Fork and Merge

  • git clone {링크}

  • git branch -a

  • git checkout -b new-feature

  • git add file

  • git commit -m “commit message”

  • git push origin new-feature

Update develop branch

  • git fetch upstream develop
  • git merge FETCH_HEAD
  • git remote add pmorigin {PM repo addr}
  • git fetch pmorigin develop
  • git merge FETCH_HEAD
